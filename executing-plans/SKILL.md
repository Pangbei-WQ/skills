# Executing Plans Skill

This skill is used to execute a previously written and approved implementation plan. It emphasizes a structured, disciplined approach with checkpoints for user review.

## When to use
- A detailed implementation plan has been created and approved.
- The task is complex enough to warrant a separate, dedicated execution phase.
- You want to ensure transparency and allow for course correction during implementation.

## Process
1. **Review Plan**: Carefully read the entire implementation plan before starting.
2. **Sequential Execution**: Follow the plan's steps in the specified order.
3. **Checkpoint Reporting**: At each designated checkpoint (or after significant steps), report progress to the user.
4. **Demonstrable Proof**: Provide evidence of work (e.g., screenshots, test output, code snippets) at checkpoints.
5. **Handle Deviations**: If implementation requires deviating from the plan, pause and seek user approval with a clear explanation of why.
6. **Final Verification**: Once all steps are complete, perform a comprehensive check against the plan's goals.

## Rules
- Do not skip steps without explicit user approval.
- Be proactive in reporting issues or blockers.
- Always provide evidence of successful step completion.
