# Receiving Code Review Skill

This skill ensures that code review feedback is handled professionally and technically rigorously. It avoids "blind implementation" of suggestions.

## Process
1. **Analyze Feedback**: Read all feedback carefully. Identify the intent and technical rationale.
2. **Technical Verification**: For each suggestion, verify if it is technically sound, consistent with the codebase, and safe to implement.
3. **Clarify Uncertainty**: If a piece of feedback is unclear, ask for clarification instead of guessing.
4. **Constructive Discussion**: If you disagree with a suggestion, provide a clear, evidence-based technical rationale for your position.
5. **Implement Valid Suggestions**: Once verified and understood, implement the changes using the project's standard workflow.
6. **Final Check**: Ensure that the review-driven changes haven't introduced regressions.

## Mindset
- Focus on technical excellence.
- Be objective and evidence-based.
- Don't agree for the sake of agreement.
- Treat every piece of feedback as a technical challenge to be solved.
