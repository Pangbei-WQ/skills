# Requesting Code Review Skill

This skill is used to formalize the request for a review of completed work. It ensures that the reviewer has all the context and evidence needed to provide high-quality feedback.

## Requirements for a Request
1. **Summary**: A clear, concise summary of the changes and their impact.
2. **Requirement Linkage**: Explicitly state how the changes satisfy the original user requirements.
3. **Evidence of Quality**: Provide proof of testing, verification, and linting.
4. **Context**: Link to relevant documentation, plans, or previous discussions.
5. **Specific Areas for Focus**: If there are parts of the code you are less certain about, point them out explicitly.

## Goal
To make it as easy and effective as possible for the user (or another agent) to review the work and ensure it meets the highest standards.
