# Finishing a Development Branch Skill

Used when a feature or bugfix implementation is complete and verified. It helps manage the transition from development to integration or finalization.

## When to use
- All implementation steps are finished.
- Tests are passing and work is verified.
- You need to integrate the work (Merge, PR) or finalize the session.

## Options to Present
The agent should present clear options to the user:
1. **Merge to Main**: Directly merge the development work into the main branch.
2. **Create Pull Request**: Prepare the work for a PR, including a summary of changes and verification evidence.
3. **Clean Up**: If the work was experimental or a temporary branch was used, offer to clean up resources.
4. **Manual Integration**: Provide instructions for the user to integrate the work themselves.

## Requirements
- Summary of work completed.
- Evidence of passing tests and manual verification.
- Identification of any remaining debt or follow-up items.
